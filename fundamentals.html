I. Mini-Machine: Humidity and Temperature Monitoring (Final Project)

A. Problem definition and real-life relevance.

Monitoring Humidity and Temperature is important in real life, because high temperatures combined with low humidity can damage equipment, plants, or stored materials. For example in greenhouses, server rooms, laboratories, or storage facilities.

This Mini-Machine simulates a real monitoring system. It checks humidity and temperature, alerts when conditions are dangerous, and stops to prevent damage. It also helps show how loops, conditions, and base cases work in python.

B. Mini-Machine Code

import random

import time

# Thresholds

MAX_TEMPERATURE = 35.0 # Celsius

MIN_HUMIDITY = 30.0 # Percentage

system_active = True # This flag keeps the loop running until the base case occurs.

# Function to check Alert conditions

def check_conditions(humidity, temperature):

return humidity < MIN_HUMIDITY and temperature > MAX_TEMPERATURE # Returns True if Humidity is too LOW and Temperature is too HIGH

print("System started. Monitoring humidity and temperature...\\n")

while system_active:

# Simulate sensor readings

humidity = random.uniform(20, 50) # Humidity between 20% and 50%

temperature = random.uniform(25, 40) # Temperature between 25C and 40C

# Print current readings

print(f"Humidity: {humidity:.1f}% Temperature: {temperature:.1f}°C")

# Check for alert

if check_conditions(humidity, temperature):

# Base case reached: Stop the machine when critical conditions are met

print("ALERT! Humidity too low and temperature too high!")

print("System stopped due to alert.")

system_active = False # Stops the Loop

break # Stop reading

# Wait a bit before next reading

time.sleep(2)

C. Explanation of the Code

The code uses a while loop that continues running as long as the system_active flag is True. Inside the loop, it simulates reading humidity and temperature values using random numbers. It then checks if the humidity is below the minimum threshold and the temperature is above the maximum threshold using the check_conditions function.

If both conditions are met, it triggers an alert, prints a message, and sets system_active to False, which stops the loop (this is the base case). The loop also includes a sleep function to pause for 2 seconds between readings, simulating real-time monitoring.

D. Reflection on challenges and how they were resolved

The first problem was when I first ran the code, the humidity and temperature were just % and C without actual numbers. The solution was to simulate realistic values using a random number generator for both humidity and temperature. By doing this, the program can display meaningful readings even without hardware. The lesson I learned is that simulating data is useful for testing and ensures that the program can be demonstrated correctly before connecting real sensors.

The second problem was when I tried stopping the program by typing “STOP”, but it did not work. This happened because the program was continuously running and never waited for user input during the monitoring loop. The solution was to set an alert condition that automatically stops the system when the humidity is too low and the temperature is too high. The lesson learned is that monitoring systems should have automatic stopping conditions rather than relying on user input, which makes the system safer and more like a real-life automatic control.

When the alert was sent, the program did not stop and kept reading values continuously. This is because there is no base case to terminate the loop . The solution was to add a base case in the code that sets the system flag to False and breaks the loop once the alert condition is reached. The lesson is that in continuous loops, defining a clear base is essential to prevent infinite execution and ensure that the system stops safely when necessary.

def is_prime(num):

if num < 2:

return False

for i in range(2, int(num**0.5)+1):

if num % i == 0:

return False

return True

def display_primes(start, end):

primes = []

for num in range(start, end+1):

if is_prime(num):

primes.append(num)

return primes

count += 1

return fib_sequence
